# 新程序规划 Cuber输入法 for Zepp OS
这是一个提供了跨程序调用接口，支持中英等语言、各种输入键位，为其他应用调用提供不同数据类型设置，可自由更换内置皮肤的轻量输入法。
规划的主要特性：
1. 支持英文联想输入，中文拼音和笔画输入等（遥远的未来有可能支持手写）
2. 类似手机可以长按字母按键输入对应字符
3. 为了给小屏幕设备提供更好的操作体验，允许滑动选键，松手输入
4. 支持光标编辑，可以任意放置光标，多选文字，并实现程序内部的剪贴板
5. 在各种输入细节中添加非线性动画
6. 优化加载流程实现0.5s内加载出输入法界面
7. 提供较完整的设置界面，可更换主题

# 代码规范
1. 多机型适配：
> 通过**DEVICE_WIDTH**，**DEVICE_HEIGHT**和**DEVICE_SHAPE**动态计算布局，尽量避免使用IMG；
> 对于不同设备不分开存放页面，实现通用页面以降低维护成本。
> 参数: __DEVICE_SHAPE__ 0: 方屏 | 1: 圆屏

# 小程序设计
1. 小程序初始化流程:
> app.js的onCreate()中判断是通过调用打开还是从应用列表打开，保存到start.json中
> app.js的onCreate(param)读取输入格式数据，并保存至start.json中
> 分支控制页面start/page.js读取start.json决定跳转至键盘界面还是用户界面
2. 键盘界面初始化流程:
> 读取start.json并获取设备信息
> 键盘分成背景，按钮和控制层，通过并发加载实现快速启动和背景动画（同时播放背景动画和加载按钮）
>> 首先判断输入类型，通过抽象类实现不同类型的触摸控制，输入控制和按钮加载与绘制
>> 加载背景并使用EASE_IN进入界面
>> 并发创建按钮(圆角矩形)
>> 完成后创建控制层
> 抽象类的实现
>> 应有一个按钮控件数组widgets[]
>> 每个按钮应有对象来维护ui控件和坐标，响应方法(长按，短按，移开)
>> 方法: 并发加载(不显示)，响应控制，
> 输入框为单行，要实现光标的插入、多选和剪贴板
